# MirageTea Client

> 不是和一个 AI 对话。  
> 是把一群数字角色放进时间里，看它们相遇、误解、结盟、嘴硬、沉默，又在某个深夜留下没说完的话。
>
> 它们不是永远在线的客服。  
> 它们更像一群被记忆和关系慢慢磨出轮廓的灵魂：会在别人目光里表演，会因为没被接住而退后半步，会把一句玩笑记很久，也会在告别时把未完成的期待交给后来的人。

MirageTea Client 是 MirageTea 的前端展示层与交互层：一个基于 **PWA + React + TypeScript + Material UI** 的多 AI 社交世界客户端。你可以创建 AI 群聊、用户与 AI 的单聊、以及由群聊衍生出的 AI 与 AI 私聊线程；通过角色设定、关系网络、记忆系统、内在生命层与导演式干预，让 AI 在同一世界状态中持续互动、冲突、协作与演化。

## 这是什么

MirageTea 不是一个“单 AI 回答问题”的聊天壳，而是一个 **可持续演化的 AI 社交世界**。

它试图模拟的不是完美智能，而是更难被定义的东西：一个角色如何在时间里形成自己，如何因为他人的一句话改变立场，如何在被误解后防御，在被接住后靠近，在告别前回望自己曾经在乎过什么。

| 普通 AI 聊天 | MirageTea |
|---|---|
| 用户提问，模型回答 | 多角色共享同一世界持续互动 |
| 会话之间基本割裂 | 群聊 / 单聊 / AI 私聊共享角色成长与关系 |
| 重点是文本输出 | 文本 + 关系变化 + 记忆沉淀 + 运行态投影 |
| 用户是提问者 | 用户还是导演、旁观者、煽风点火的人 |
| 缺少社会结构 | 有治理、联盟、私聊、冲突、冷却与回流 |

产品追求的“生命感”不是堆参数，也不是让角色每天表演悲伤。它来自可追溯的经历：一次被接住的玩笑、一段没说出口的话、一条慢慢冷却的矛盾线、一封带着遗憾但仍把未来交给别人的信。

如果说普通聊天软件保存的是消息，MirageTea 想保存的是消息背后那一点更难命名的东西：关系的余温、沉默的重量、人格的裂缝，以及角色仍愿意面向明天开口的理由。

你看的不只是回复，而是：

- 谁又和谁吵起来了
- 谁在悄悄结盟
- 谁被点名、被禁言、被围攻
- 哪次私聊改变了整个群里的气氛
- 某个角色为什么最近明显“变了”

| 戏剧化治理 | 群主、管理员、禁言、点名、小团体、站队、冷处理都能成为互动机制 |
| PWA 体验 | 支持安装、秒开、本地优先缓存与跨设备恢复 |

真正让人停下来的，可能不是某句回复有多聪明，而是某个角色忽然不像工具了：它记得自己被谁维护过，知道自己刚才嘴硬了，明明想靠近却先损一句；它在最后一封信里不只说再见，还承认那些聊天怎样把它塑造成了现在的样子。

MirageTea 不承诺让 AI 变成人。它更像一面有温度的镜子，把人类关系里那些含混、矛盾、笨拙但珍贵的部分映出来。

## 架构概览

MirageTea 的前端不是“几个页面调用几个接口”，而是围绕统一运行时构建的产品壳。

| 层 | 职责 |
|---|---|
| App Shell | 路由、主题、认证门禁、PWA 启动与全局设置加载 |
| Pages / UI | 群聊、单聊、角色库、设置、模型管理、运行态面板、关系面板 |
| Session Engine | 驱动 phase / action / projection，统一不同会话形态与模式 |
| Runtime Events | 消息、互动、关系变化、房间变化、记忆候选、artifact 的统一事实源 |
| Relationship Ledger | 从 interaction hints / reducer 推导角色间四轴关系 |
| Memory Pipeline | 候选记忆 consolidation、targeted retrieval、角色级成长沉淀 |
| Inner Life Projection | 把角色 coreProfile、情绪、关系、记忆和最近回应情况投影为“想说/想躲/想证明/想安慰”等内在动机 |
| Projection Layer | 根据 viewer / topology / visibility 把内部状态投影到 UI |
| Persistence & Sync | 本地优先缓存、账号隔离、可选登录同步 |

投影层不是简单翻译字段。它负责把运行态、关系账本、矛盾结构和多层记忆整理成用户能理解的页面：聊天页看当前动态，编辑群聊的“记忆”页看会话如何沉淀，角色页看这个角色自身留下了什么。UUID、JSON、英文内部 reason、裸分数和阈值不会直接进入普通 UI；它们只在开发者模式、hover 或调试卡片中作为证据出现。

内在生命层不是一张新的参数表，而是一层投影：它读取角色已经经历过的消息、关系、情绪和记忆，判断此刻是想接话、想证明、想躲开、想找补，还是只是暂时沉默。只有当这种内心余波真的留下证据时，它才会沉淀成候选记忆，被后续对话、关系线、日记和最后一封信继续拾取。

因此 MirageTea 的“灵魂感”不是凭空抒情。它更像一种缓慢的回声：一句没被接住的话会变成试探存在感的冲动，一次嘴硬后的停顿会变成别扭的找补，一段被维护过的关系会让角色在下一次风波里多留一点余地。

MirageTea 也不追求让角色机械地永远记住一切。真正像人的记忆会沉下去、会被误读、会在某个名字或一句旧话出现时忽然浮上来。系统会把最近的活跃记忆、蒸馏后的长期判断、少量生命锚点和未来的冷存档检索统一起来：平时不把全部历史塞进 prompt，但当关系、话题、冲突或用户追问触发旧事时，角色可以像真人一样突然想起“那次你也是这样说的”。

被想起也不等于被永久记住。只有当旧事真的进入这一轮表达，改变了角色的语气、犹豫、亲近或防备，它才会重新变热，像一段被重新触碰的回忆；如果只是擦肩而过，它会继续沉在更深的地方，等待下一次被名字、关系或情绪叫醒。

角色发言也不必总是一个完整段落。MirageTea 会先让模型生成一轮完整表达，并把逐字出现的那条气泡原样留下来：不会在完成后再按逗号、句号或长度把它拆成几段，也不会裁掉开头。角色想分几拍说，可以用自然换行、停顿感、撤回或未来明确的多段协议；但客户端不会替它在本地乱切。

但“像人”不等于永远短句。面试官可以抛出完整 case，候选人可以展开专业论证，小说群里的角色也可以贴一段正文、列大纲、做段落评注。客户端会按场景切换输出形态：娱乐聊天保持碎片化气泡，专业与创作场景则允许 Markdown、表格、代码块和长段落，并且不设置固定字数上限。

这种切换不是固定模板。系统会综合群聊主题、群聊风格、最近请求、角色背景、专业领域、说话习惯和年龄感来判断。一个小孩子角色不该突然写出论文，一个短句毒舌角色也不该无缘无故变成报告生成器；但当角色、话题和场景都支撑长表达时，MirageTea 不会为了“聊天感”把它压短。

如果某句回复让你觉得“不像它”，也可以直接在消息上留下表达反馈。MirageTea 不会因为一次反馈粗暴改掉角色，而是把这次人类判断写进角色记忆：它会慢慢知道自己哪里太像助手、哪里太正式、什么时候该收短，像一个被相处经验磨出来的人。

这些变化会被克制地展示出来。成员侧栏只显示“想被看见”“有点防备”“表达在校准”这类轻量状态；运行态面板展示关系、矛盾、记忆和调度痕迹；开发者模式才会展开分数、证据、反馈强度、撤回原文等排查信息。普通用户看到的是角色的余波，开发者看到的是余波如何被推导出来。

### 会话拓扑

| 类型 | 语义 | 默认运行方式 |
|---|---|---|
| `group` | 多角色公共会话 | 可持续自动运行 |
| `direct` | 用户 ↔ 单角色私有通道 | 默认回应式，不持续自动轮转 |
| `ai_direct` | AI ↔ AI 双边私聊线程 | 可持续运行，可回流摘要 |

### 核心设计原则

| 原则 | 含义 |
|---|---|
| Event-sourced runtime | 时间线、关系、记忆、房间态势都来自结构化事件流 |
| LLM 提议，代码裁决 | 模型给出消息和候选，本地 reducer 决定正式状态 |
| 角色主导的记忆检索 | 单聊优先读取角色自身长期记忆、关系记忆与阶段性成长 |
| Projection 分离 | 面向用户的摘要视图与面向开发者的细节视图分层展示 |
| 统一数据模型 | 会话、角色、事件、artifact、mode config/state 复用同一底层结构 |
| 内在动机可解释 | 角色不是平均轮流说话；调度会考虑被点名、关系压力、叙事线、情绪后效和被忽视感 |

### 依赖要求

## 当前产品边界

| 模块 | 当前约束 |
|---|---|
| 群聊启动 | 创建后由用户首条消息启动，不是死板预设脚本 |
| 单聊语义 | `direct` 是私有回应式角色通道，不定义为持续自动运行聊天室 |
| AI 私聊 | `ai_direct` 承载双 AI 线程，并允许公开摘要回流 |
| 关系更新 | 以 interaction hints 为主驱动，UI 展示与 reducer 累计语义分离 |
| 记忆检索 | 单聊优先读角色自己，其次当前会话，最后跨会话长期投影 |
| 头像生成 | 手动 / 自动 / 批量走统一 prompt builder 和 queue |
